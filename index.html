Questions fréquentes

Mes données sont-elles sauvegardées si je change de navigateur ? +
Oui. Tes données sont hébergées sur Supabase (cloud européen), pas dans ton navigateur. Tu peux y accéder depuis n'importe quel appareil avec ton email et mot de passe.
Comment accéder à mon espace depuis un autre appareil ? +
Tu accèdes à ton espace depuis n'importe quel navigateur avec ton email et ton mot de passe. Toutes tes données sont synchronisées en temps réel via Supabase.
Puis-je résilier à tout moment ? +
Oui, sans frais ni préavis pour le plan mensuel. Pour le plan annuel, l'accès reste actif jusqu'à la fin de la période payée.
Est-ce que l'outil calcule la TVA ? +
L'outil raisonne hors taxes par défaut (comme un tableau de rentabilité professionnel). Les prix affichés sont nets — CEO Room ne collecte pas la TVA.
Que se passe-t-il après le paiement ? +
Stripe te redirige directement vers le formulaire de création de ton espace CEO Room. Tu saisis ton prénom, ton email et un mot de passe — et tu accèdes immédiatement à ton cockpit.

Guide de configuration

Suivez ces étapes pour connecter Supabase (base de données) et Stripe (paiements). Durée estimée : 30 minutes.

1
Créer un projet Supabase
Rendez-vous sur supabase.com → New Project. Choisissez la région Europe (Frankfurt) pour la conformité RGPD. Notez votre Project URL et votre clé anon/public.
https://app.supabase.com → New Project → EU (Frankfurt)
💡 Le plan Free Supabase suffit pour démarrer (500 Mo, 2 projets).
2
Créer les tables SQL
Dans Supabase → SQL Editor, exécutez ce script pour créer la structure des données :
-- Table des clientes
CREATE TABLE clients (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  admin_id TEXT NOT NULL,
  prenom TEXT NOT NULL,
  nom TEXT NOT NULL,
  email TEXT,
  company TEXT,
  password_hash TEXT,
  data JSONB DEFAULT '{}'::jsonb,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Index pour accès rapide
CREATE INDEX idx_clients_admin ON clients(admin_id);

-- Row Level Security
ALTER TABLE clients ENABLE ROW LEVEL SECURITY;
CREATE POLICY "admin_access" ON clients FOR ALL USING (true);
3
Configurer les variables Supabase dans l'outil
Dans le code JS de l'outil (cherchez SUPABASE_URL et SUPABASE_KEY), remplacez les valeurs par celles de votre projet Supabase :
const SUPABASE_URL = 'https://VOTRE_REF.supabase.co';
const SUPABASE_KEY = 'votre_anon_key_ici';
const ADMIN_ID = 'wwp_admin'; // identifiant unique de votre compte
4
Créer un compte Stripe
Rends-toi sur stripe.com → crée ton compte → active le mode Live. Crée 2 produits : CEO Room Mensuel (29 € récurrent mensuel) et CEO Room Annuel (290 € récurrent annuel). Notez les Price IDs (price_xxxxx).
⚠️ En mode test, utilisez les Price IDs test. Passez en Live uniquement avant de partager l'URL avec vos clientes.
5
Configurer les liens Stripe dans le code
Dans le code JS (cherchez STRIPE_LINKS), remplacez par vos liens Payment Links Stripe :
const STRIPE_LINKS = {
  mensuel: 'https://buy.stripe.com/VOTRE_LIEN_MENSUEL',
  annuel: 'https://buy.stripe.com/VOTRE_LIEN_ANNUEL'
};
6
Configurer la Success URL Stripe (redirection post-paiement)
Après paiement, Stripe redirige automatiquement la cliente vers le formulaire de création de son espace (via la Success URL configurée dans le Payment Link). Dans le dashboard Stripe → Payment Links → ton lien → Confirmation page, active "Redirect to URL" et entre : URL_DE_TON_OUTIL/#signup. La cliente arrive directement sur le formulaire d'inscription sans aucune action de ta part.
💡 Exemple de Success URL : https://ceoroom.netlify.app/#signup
7
Héberger l'outil (optionnel)
Pour avoir une vraie URL (ex: app.womenwithaplan.fr), hébergez le fichier HTML sur Netlify (drag & drop, gratuit) ou GitHub Pages. Connectez votre domaine dans les paramètres DNS.
netlify.com → New Site → Drop your HTML file → Done ✓
💡 Netlify offre un certificat SSL gratuit et un CDN mondial — parfait pour ce type d'outil.
